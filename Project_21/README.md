# Проект 21: Дообучение языковой модели (Text-to-SQL) с помощью QLoRA

## 📌 Обзор проекта
Данный проект представляет собой полный цикл дообучения большой языковой модели (LLM) для перевода запросов на естественном языке в исполняемые SQL-коды. Проект выполнен с использованием техники параметрически эффективного дообучения **QLoRA** (4-битная квантизация).

Базовая модель: `Qwen/Qwen2.5-3B-Instruct`
Домен: База данных e-commerce магазина (клиенты, заказы, товары).

## 🎯 Мотивация выбора задачи
Задача генерации SQL-запросов (Text-to-SQL) была выбрана по нескольким причинам:
1. **Объективность оценки:** SQL-код имеет строгий синтаксис. Легко проверить, является ли сгенерированный ответ рабочим кодом или галлюцинацией.
2. **Сложность для базовых моделей:** Без дообучения модели часто добавляют лишний текст (например, "Вот ваш запрос..."), используют текстовый поиск там, где нужны диапазоны, или игнорируют предоставленную схему (контекст таблиц). Дообучение заставляет модель выводить *исключительно* валидный SQL.

## 📊 Подготовка данных (Часть 1)
Был создан кастомный датасет в формате `JSONL`.
* **Обучающая выборка (train.jsonl):** 90 примеров.
* **Тестовая выборка (test.jsonl):** 20 примеров.

**Особенности датасета:**
* Покрыты различные подтипы SQL: агрегации (COUNT, SUM), JOIN-соединения, фильтрация по датам, группировки (GROUP BY).
* Включены *негативные примеры*: если пользователь просит изменить данные (UPDATE/DELETE) или спрашивает то, чего нет в базе (погода), модель выдает корректный отказ ("Ошибка: ...").
* Проведена строгая валидация на отсутствие утечек данных (Data Leakage) между Train и Test.

## 🛠 Дообучение и оценка (Часть 2)
Обучение проводилось с использованием библиотеки `peft` и `trl` (SFTTrainer) на видеокарте уровня Tesla T4 (Google Colab).

* **Метод:** QLoRA (Rank=8, Alpha=16, target_modules=["q_proj", "v_proj"]).
* **Оптимизация памяти:** 4-bit NormalFloat (NF4) квантизация базовой модели. Формат вычислений жестко зафиксирован в `Float16` для аппаратной совместимости.
* **Метрики и результаты:** Во время обучения Training Loss снизился с **1.90 до 0.34**.
* Сравнительная оценка (Inference) на тестовой выборке показала, что дообученная модель генерирует строгий SQL без текстового мусора и лучше справляется с логикой (например, корректное использование диапазонов дат вместо `LIKE`), в то время как базовая модель часто игнорирует формат или применяет неоптимальные SQL-операторы.
