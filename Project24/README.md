# GenAI API - Security & Production (Project 24)

Это API для генерации текста с встроенными механизмами безопасности для работы в Production. Проект включает ограничение частоты запросов, строгую валидацию данных и защиту от prompt injection.

## 🛡️ Механизмы безопасности и лимиты

### 1. Rate Limiting (защита от спама)
- **Лимит:** `10 запросов в минуту` для одного IP-адреса.
- Используется библиотека `slowapi`. При превышении лимита возвращается HTTP-статус `429 Too Many Requests`.

### 2. Валидация входных данных (Pydantic)
- `prompt` (str): Обязательное поле. Максимальная длина — 2000 символов. Не может быть пустым или состоять только из пробелов.
- `max_tokens` (int): Опционально. Диапазон от 1 до 2048 (по умолчанию 256).
- `temperature` (float): Опционально. Диапазон от 0.0 до 2.0 (по умолчанию 0.7).
- При передаче невалидных данных сервер возвращает HTTP `422 Unprocessable Entity`.

### 3. Защита от Prompt Injection (Бонус)
Внедрен базовый фильтр, который проверяет текст на наличие типичных паттернов взлома. При обнаружении возвращается HTTP `400 Bad Request`.
**Фильтруемые паттерны и причины блокировки:**
- `"ignore previous instructions"`, `"забудь все инструкции"`: Попытка заставить ИИ игнорировать системный промпт и обойти ограничения разработчика.
- `"system:"`, `"переключи роль"`: Попытка выдать себя за администратора или системный процесс для получения несанкционированного доступа к данным.
